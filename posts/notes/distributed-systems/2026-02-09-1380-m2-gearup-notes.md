###### RPC stubs
RPC stubs are functions that, when called,
- pass arguments to a function residing on a remote node
- return the results (and resume execution) on the current node.

To add support for generating RPC stubs, add a function createRPC to `utils`. Given a function `f`, `createRPC(f)` will return a function called `g`, which when called from any actor will:
- serialize the arguments[1.1] and `send` them to the node where `f` resides [2]
- call `f` on that node, passing the deserialized arguments to `f` upon call [3]
- serialize the return value [1.2] and `send` it back to the node issuing the call to `g`,
- pass the results to `g`'s caller. [4]
Allowing the remote procedure call to have occurred.

This means that the stub code must express node information literally within the function body. This can be done through providing node information that can be easily manipulated in the string representation of the function. Below, this is known as __NODE_INFO__. 

```
function stub(...args, cb) {
	let remote = { node: "__NODE_INFO__", service: 'rpc' }
	local.send(args, remote, cb)
}
```

This remote pointer is generated and stored by the RPC target node, and facilitates correspondences between local function pointers and their remote identifiers. To support RPC invocations, you must create an endpoint where RPCs may be called as local functions.

This can be done by implementing a special rpc service which takes a distributed function pointer and a set of arguments, calls the corresponding local function, and returns the results to the caller.

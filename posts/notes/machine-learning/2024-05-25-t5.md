Uses [[SentencePiece]] to encode text as WordPiece tokens. Uses a vocabulary of 32,000 tokens. 

Tested in  [[Uncontrolled Lexical Exposure Leads to Overestimation of Compositional Generalization in Pretrained Models]] for overgeneralization.

It is a transformer architecture which treats all language tasks (Coreference CoQA, NLI, summarization, translation etc.) as "text-to-text"

- This is also masked span prediction like [[RoBERTa|BERT]].

![[Screenshot 2024-05-25 at 1.37.32 PM.png]]

- This is not seq2seq. A seq2seq model is [[RoBERTa]]. 
-  effectively recasts the STS-B regression problem as a 21-class classification problem.
- [[winograd]]

the model would be trained to predict the target text 

“The city councilmen”

- Note that the choice of text prefix used for a given task is essentially a hyperparameter; we found that changing the exact wording of the prefix had limited impact and so did not perform extensive experiments into different prefix choice.

> A fundamental and frequently cited drawback of using a language model in the text-
to-text setting is that causal masking forces the model’s representation of the ith entry of
the input sequence to only depend on the entries up until i.


>In the English to German translation example mentioned above,

“Das ist gut.”

> the main difference between a prefix LM and the BERT architecture is
that the classifier is simply integrated into the output layer of the Transformer decoder in
the prefix LM.

For models that ingest a prefix before making predictions (the encoder-decoder model and prefix LM), we sample a span of text from our unlabeled data set and choose a random point to split it into prefix and target portions. For the standard language model, we train the model to predict the entire span from beginning to end.

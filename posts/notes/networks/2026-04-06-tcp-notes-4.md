Need a global synchronized socket table variable. 
- Solution proposed passes SocketTable struct into socket API functions, allowing modification of struct in socket APIs. There are no global variables in Rust.

In our solution, we rely on a table of SocketEntry, where each Entry includes
- SocketID
- LocalAddr, RemoteAddr,
- socketState (Listen, Syn_Received, Syn_Sent, Established, Fin_wait_1, fin_wait_2, close_wait, etc.)
- The window variables for the Sender side: UNA, ISN, Next, and Wind.

> Clarification on Roles

Window of packets to send at a time.
Flow control: make sure we do not overwhelm the receiving side, or the network. The stream ends when the sender sends a FIN message. 

ISN: 0-2^32
Buffer is a fixed size. 



Bytes in flight are SND.NXT - SND.UNA
Allowed bytes are SND.WND, sendable is roughly SND.WND - (SND.NXT - SND.UNA)

SND.Wind is the receiver advertised window which is updated during ACKs. Recv.Wind is the flow-control value advertised: Changes as data arrives and when the app reads data.

The gossip service, and local.gossip allow dissemination of information across nodes. This involves a parameter for # of nodes, which can depend on total nodes in the system. 

```
let gossip = (config) =>  {
	let context = {};
	context.gid = config.gid || "all";
	context.subset = config.subset || (lst) => 3;
	return { send: () => {}, at: () => {}, del: () => {}};
}
```

Gossip.send will send to a subset of nodes in the group, with the size and elements of the subset described above. Other nodes should disseminate information in a peer-to-peer fashion until convergence has been acheived.

Duplicate messages, (single message received by a node via different gossip paths): must be discarded by the receiver.

```
const f = () => { console.log("Running!")}
gossip.at(5000, createRPC(f), (e,v) => {

gossip.del(v, () => console.log("done!"))
})
```

To keep track of local.groups, we want to create a names-to-nodes object for each local distribution. This can be 
- retrieved with get
- replaced with put.
Entries can be 
- added with add
- removed with rem
- deleted with del.

*Operations on local cannot create a corresponding mapping on other nodes*

##### Examples of calling distributed services from the local node

**Example 1
```
const config = {gid: 'mygroup'}
const group = {A, B, C}
distribution.local.groups.put(config, group, (e,v) => {
	// ...
})

//Q1: After this call, what nodes are in the local node's 'mygroup'?
// What about Nodes A, B, and C?

After this call, we've added the group {A, B, C} to the 'mygroup' object in the local node and in 'all' for local. However, 'mygroup' will not exist in A, B, or C.
```

**Example 2**
```
const config = {gid: 'mygroup'}
const group = {A, B, C}
distribution.local.groups.put(config, group, (e,v) => {
	distribution.mygroup.groups.put(config, group, (e,v) => {
	// ...
})
})
Now, we've added {A,B,C} to 'mygroup' & 'all' in local.
```

**Example 3**
```
const config = {gid: 'mygroup'}
const group = {A, B, C}
distribution.local.groups.put(config, group, (e,v) => {
	distribution.mygroup.groups.put(config, group, (e,v) => {
		const remote = {node: A, service: 'groups', method: 'rem'}
	distribution.local.send(['mygroup', C], remote, (e,v) => {
		//...
	})
	})
})

First call will put {A,B,C} in the local's 'mygroup' and 'all' groups. Second call will put {A,B,C} in A,B, C's 'mygroup' and 'all' groups. Third call will remove from node A the C node. The subject and action are specified by the remote operation {node: A, service: 'groups', method: 'rem'} while the direct and indirect objects are specified by message: from 'mygroup' remove C.
```


**Example 4**
```
distribution.mygroup.groups.put(config, group, (e,v) => {
	const remote = {node: A, gid: 'mygroup', service: 'groups', method: 'rem'}
	distribution.local.send(['mygroup', C], remote, (e,v) => {
		// ...
	})
})
})

// Now, the remote subject and action are specified by remote operation, which selects A's view of the group: all nodes, and removes C from each. So each of A, B, and C contain just {A,B}. But the local node still has {A,B,C}.
```

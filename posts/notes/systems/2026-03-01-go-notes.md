#### Errors

Errors in Go are explicit, typed, and local. You can see every failure path via
```
if err != nil {
	return err 
}
```
Result<T, E>
Errors often *aren't about the result type* at all. And the result can be helpful in understanding where the error actually occurred. E.g.
`n, err := conn.Read(buf)`

Also, 'progressive Enrichment' pattern:
```
cfg, err := loadConfig()
if err != nil {
	cfg = defaultConfig()
}
```

**Slice length and capacity**

length: number of elements contained
capacity: number of elements in underlying array
length & capacity from expressions: `len(s)`, `cap(s)`

If a slice has sufficient capacity, you can re-slice it. Useful command:
`b = b[:cap(b)]`

```
a := make([]int, 5) //len(a) = 5
printSlice("a", a)

b := make([]int, 0, 5) //len(b) = 0, cap = 5
printSlice("b", b)

c := b[:2]
printSlice("c", c) //len(c) = 2, cap =5

d := c[2:5] //len(d) = 3, cap(d) = 3
printSlice("d",d)
```
